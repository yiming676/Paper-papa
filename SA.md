# Study Assistant 需求沉淀

## 产品目标

Study Assistant 是一个面向陌生领域论文学习者的本地优先工具。用户上传一篇论文 PDF 后，系统生成第一层结构化学习笔记，并把正文中首次出现的关键科研词汇变成可点击概念入口。用户点击概念后，再按需调用 API 生成下一层概念页，概念页继续支持必要前置概念递归跳转。

工具默认面向小白和跨领域读者，但解释必须保持科研严谨性。当前发布形态优先支持本地下载运行，用户自行配置 OpenAI-compatible API Key；后续可以扩展为多人网页服务。

## 第一层学习笔记固定 8 部分

不管用户选择什么模型或 API，第一层学习笔记都必须尽量按下面 8 个核心部分生成。若模型、API 或 PDF 抽取无法满足要求，系统展示可用内容，并在页面 warning 中明确说明限制来源。

1. 领域定位
   - 一级领域。
   - 二级方向。
   - 本文解决的具体问题链条。

2. 研究问题
   - 作者想解决什么问题。
   - 这个问题为什么重要。
   - 已有方法为什么不够好。
   - 输入、输出和评价标准。

3. 核心概念与术语如何串起本文方法
   - 不再把术语作为独立卡片堆叠展示。
   - 用叙述型正文解释基础概念、方法概念、实验概念、数学概念、工程概念如何服务于本文主线。
   - `learning_keywords` 只作为内部链接元数据；这些词必须自然出现在对应正文段落中，供前端在首次出现处加链接。

4. 公式拆解
   - 核心公式优先。
   - 每个公式解释：算什么、符号含义、输入变量、输出结果、服务模块、代码步骤、去掉后的影响、作者为什么这样设计。

5. 方法整体 pipeline
   - 本次先用文本流程图呈现，后续再升级 Mermaid。
   - 说明数据来源、模块连接、每个模块输入输出、哪些是已有方法、哪些是本文创新、哪些是工程实现。

6. 本文和已有工作的关系
   - baseline。
   - 继承或改进对象。
   - 与前人最大区别。
   - 方法对比：核心思想、优点、缺点、本文如何改进。

7. 实验逻辑
   - 数据集、比较方法、指标。
   - 主实验、消融实验、可视化结果、失败案例。
   - 判断这些指标是否足够支撑作者结论。

8. 假设与局限
   - 依赖前提。
   - 适用和失败场景。
   - 数据集偏差、指标偏差、计算成本、复杂预处理、复现风险、隐藏工程技巧。

## PDF 图表处理

- 当前不做 PDF 图片截图保存。
- 当前不解析表格单元格，也不展示抽取表格。
- 系统只从 PDF 文本中识别 Figure、Fig.、Table、图、表 等图表引用及附近 caption/context。
- 后端将这些线索保存为轻量 `paper_references`，字段包括 `type`、`label`、`page`、`caption`、`context`、`section`、`reason`。
- 学习笔记只用文字提示用户“参见 Figure X / Table Y”，用户自行回论文原文查看。

## 点击和递归规则

- 第一层学习笔记正文中的术语、参数、公式只给首次出现的实体加链接。
- 不在第一层单独展示术语卡片；用户想理解关键词时，点击正文中的关键词进入概念页。
- 第一层文档生成时只创建可点击概念链接，不预生成概念页。
- 用户点击 `/concepts/{id}` 后，后端才调用 API 生成该关键词的下一层学习笔记。
- 概念页必须强关联原论文上下文，使用首次出现附近文本、论文标题、父概念路径和相关 Figure/Table 引用线索；禁止泛百科式解释。
- 概念页正文中的新专业词汇也只在首次出现处加链接。
- 从第一层论文概念开始，递归跳转上限为 10 层。
- UI 必须显示当前层级，例如“第 3/10 层”。
- 第 10 层仍可展示解释，但不再继续自动展开下一层，并提示已到递归上限。

## 失败提示规则

当系统无法满足模板时，不静默降级。需要明确告诉用户更可能是哪类问题：

- 模型能力限制：模型没有按 schema 返回完整内容，或无法完成公式、实验、局限分析。
- API 返回问题：接口返回无效 JSON、空内容、超时、HTTP 错误或字段缺失。
- PDF 抽取问题：文本或公式无法从 PDF 中可靠抽取；当前不做图片截图和表格单元格抽取。
- 配置问题：API Key、Base URL、模型名、token 上限或跨域配置异常。

默认策略是“带 warning 展示”：展示模型生成的可用部分，但在缺失位置和整体 warning 中提示限制来源。

## 当前实施边界

- 本地优先。
- 用户自行配置 OpenAI-compatible API Key。
- 不新增多用户账号、权限隔离或统一模型计费。
- pipeline 图本次先用文本流程图，后续再升级 Mermaid。
- 图表只保留文字引用线索，不展示 PDF 图片或抽取表格。
