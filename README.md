# Recursive Paper Study Tool MVP

一个本地可运行的“递归式论文学习工具”最小可实现版本。

上传一篇英文 AI / CS 论文 PDF 后，系统会完成这条最小闭环：

1. 解析 PDF 文本
2. 转成结构化 markdown
3. 抽取术语、参数、公式
4. 只给首次出现的实体加链接
5. 进入概念页查看固定模板解释
6. 继续递归展开必要前置概念
7. 标记概念为 `unknown`、`learning`、`mastered`
8. 在 mastered 页面查看已学会概念

## Tech Stack

- Frontend: Next.js, TypeScript, Tailwind CSS
- Backend: FastAPI, SQLAlchemy, Pydantic
- Database: PostgreSQL
- PDF parser: PyMuPDF
- Formula rendering: KaTeX
- LLM access: OpenAI-compatible REST API, configurable by env

## Core Design

- 前后端分离，前端只通过 REST API 调后端。
- 所有 LLM 任务封装在 `backend/app/services/llm_service.py`。
- Prompt 独立放在 `backend/app/prompts/`，后续可直接替换。
- 如果没有配置 `OPENAI_API_KEY`，系统会退回启发式逻辑，保证 MVP 能跑通。
- 概念递归深度限制为 4 层。
- 同一实体在同一篇文档内只标注首次出现位置。

## Local Development Without Docker

如果你本机没有 PostgreSQL，项目也可以先用 SQLite 跑本地 MVP。
这只是为了本地开发便利，Docker 方案仍然默认使用 PostgreSQL。

### Recommended one-command start on Windows

From the project root, run:

```powershell
.\start-local.cmd
```

Or, if you prefer PowerShell directly:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\start-local.ps1
```

This starts both services in separate service windows, creates `.env` from `.env.example` when needed, and installs backend/frontend dependencies when they changed.

After startup, open:

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8000`
- Swagger: `http://localhost:8000/docs`

Press `Ctrl+C` in the launcher window to stop both services.

To stop the local services from another terminal, run:

```powershell
.\stop-local.cmd
```

### LLM report generation and fallback

The study report is now generated by the configured OpenAI-compatible LLM API. Local code only extracts PDF text, tables, and figure screenshots, then sends that evidence to the model.

If the LLM API is not configured or returns invalid JSON, parsing fails with a clear backend error instead of silently showing a heuristic fallback report. Check these variables in `.env`:

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_CHAT_COMPLETIONS_PATH`
- `MODEL_NAME`
- `LLM_MAX_TOKENS`

`ALLOW_LLM_FALLBACK=false` is recommended for real study notes. Set it to `true` only when you deliberately want a rough offline demo.

For an existing uploaded paper, open its document page and click “重新解析当前文档” to regenerate the report with the current pipeline.

## MVP Workflow

1. 打开上传页 `/`
2. 上传 PDF
3. 前端自动串联调用上传、解析、标注
4. 跳转文档页 `/documents/{id}`
5. 点击首次出现实体，进入概念页 `/concepts/{id}`
6. 点击 `Expand prerequisites`
7. 点击状态按钮标记学习状态
8. 在 `/mastered` 查看已学会概念

## Simplifications In This MVP

- 默认单用户，不建 `users` 表
- 没有做多文档知识融合
- 没有做向量检索
- 没有做复杂公式 AST 级分析
- 没有做复杂术语消歧，使用基础归一化与缩写映射
- 若 LLM 不可用，使用启发式抽取保证流程可运行
